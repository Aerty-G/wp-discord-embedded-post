Plugin Overview

This plugin provides comprehensive integration with Discord, allowing you to:

  • Create dynamic variable options with different processing modes
  • Configure Discord connection via webhook or bot
  • Design rich embed messages with buttons
  • Set category-specific notification settings
Tip: Start by configuring your Discord connection settings first.

Variable Placeholder Options Configuration

Where can I use this placeholder?

Almost all parts that can be filled can use this.

For more details where you can put it, please see the following.

  • Main Message
  • Embedded In Author Name Section
  • Embedded In Title Section
  • Embedded In Description Section
  • Embedded In Name And Value Inside Fields Section
  • Embedded In label Inside Button Components Section
  • Embedded In Text Inside Footer Section

Template Placeholders

Create template names that can be used as placeholders in your embeds and the main message in category settings:

${template_name}$

Processing Modes

  • Single: Uses the first meta key's value
  • Combine: Joins all values with separator
  • Connect: Uses first value as post ID to get second value

Dynamic Meta Placeholders

Are you too lazy to set variables in the available settings? You can also directly do a 'direct get' in this way.

${get_post_meta => [mode],[keys],[separator]}$

Here you can set to fetch meta in a similar way to setting it in the settings but here you set it 'directly'.

  • Mode: like the explanation in above.
  • Keys: In the 'keys' section you can fill it with 1 or more meta keys, each separated by ','.
  • Separator: in the 'separator' section you can fill it with anything, because this section is the separator between each value of the meta key, but only for Combine mode.

Defaults Placeholder

Here's The List Of All Defaults Placeholder.

  • ${author}$: To Showing Author Of Current Post.
  • ${timestamp}$: To Showing Timestamp Of Current Post, in Discord Format.
  • ${permalink}$: To Showing Permalink Of Current Post.
  • ${thumbnail_url}$: To Showing Thumbnails Url Of Current Post.
  • ${post_title}$: To Showing Title Of Current Post.
  • ${post_type}$: To Showing Status Of Current Post, Like Published, Draft, Future.
  • ${post_name}$: To Showing Slug Of Current Post.
  • ${post_content}$: To Showing Content Of Current Post.
  • ${post_content:type_trim:number}$: To Showing Content Of Current Post, but the content itself will get trim base on words or letter. If Base on Words fill type_trim with 0, if by letter fill type_trim with 1.
  • ${default_tag}$: To Showing Default Tag You Set in The Default Settings.

Dynamic Post Info Placeholders

By using this you can retrieve information from posts either the current post, or a post with a certain ID, or even a post from the id that is in the meta in the current post.

${get_post_info => [id/meta_key],[info]}$
  • ID/Meta_Key: This section if you fill it with the ID of a post it will retrieve information based on the ID, but if you fill it with meta_key it will retrieve the meta value based on the current post ID then the value will be the basis for retrieving the post information, then if you leave it blank it will retrieve the ID of the current post.
  • Info: in this section you fill it with the information you want to retrieve from the post based on the ID column earlier, the list will be below.
  • post_author: To Get Post Author.
  • post_date: To Get Publish Date Of The Post.
  • post_name: To Get Post Title Slug.
  • post_content: To Get Post Content.
  • post_title: To Get Post Title.
  • post_status: To Get Post Current Status.
  • post_type: To Get Post Type.
  • post_category: To Get Post Category.
  • thumbnail_url: To Get Post Thumbnail Url.
  • permalink: To Get Post Permalink.

Dynamic Term Info Placeholders

${get_term_list => [ID/meta_key],[taxonomy_name],[separator],[before],[after],[filter]}$
  • ID/Meta_Key: If you fill it with a Numeric Number, it will be taken as the ID to Retrieve the Term later, if you fill it with meta_key, it will retrieve the meta value based on the key with the current post ID then the meta value will be used as the ID to retrieve the term, if you leave it blank, it will take the current post ID as the ID to retrieve the term.
  • Taxonomy_Name: The name of the taxonomy you will take.
  • Separator: Separator between the terms that are retrieved later..
  • Before: The string or html that is at the front of the list..
  • After: The string at the end of the list..
  • Filter: If you want to filter the results of the term list with strip_tags then please fill it with the number 1 if not leave it blank, then the list will be displayed as it is..
